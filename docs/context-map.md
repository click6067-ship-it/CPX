# CPX-AI 프로젝트 — 컨텍스트 맵 (Context Map)

> **목적:** 이 폴더의 자료·결정·맥락을 한 장으로 정리. 새 세션이나 다른 AI agent가 읽고 **바로 맥락 파악 + 근거 위치 찾기** 위한 지도.
> **최종 갱신:** 2026-06-18 · 작성: 김용하 + Claude
> **읽는 법:** 0→1을 먼저 읽으면 프로젝트가 잡힘. 세부 근거가 필요하면 2(파일)·3(데이터)에서 찾아 원문 확인.

## 0. 프로젝트 한 눈에

- **무엇:** AI로 CPX(임상수행평가) ① 사례 자동 생성 → ② 전문가 심사·개선 → ③ 가상환자 대화 → ④ 자동 채점·피드백.
- **왜:** 사례개발 1건 15시간(교수 10명 1박2일), 채점 24명×8시간, 학생 연습 파트너 부족 → AI로 해소.
- **팀(역할):**
  - 임선주 교수 = PI(총괄·의학검증·발표·교신저자)
  - 이혜윤 교수 = 공동PI(사례검증·체크리스트 타당성·임상자문)
  - 채원우 = **사례 생성 AI**(①) 총괄, 데이터 전처리
  - 조호영 = STT/TTS·시스템통합·UI·학사연동
  - **김용하(나) = 채점 로직(④)·백엔드·DB·API·보안**
  - 최수연 조교 = 데이터(사례 원본 제공)·행정
- **두 문서 = 한 프로젝트:**
  - 킥오프 회의록(260603) = **풀 시스템 20주 빌드 계획** (에듀테크 엑스포 발표 의무)
  - 사업계획서 `.hwp`(2026.4, 임선주) = **연구지원사업 framing** (CPX 교과 실증 60명, IRB, 11월 SUMMIT)
- **돈/일정:** ~2억/3년. 1차 핵심 마일스톤 = 8월 말까지 AI 생성 사례 50개 중 30개+ 교수 승인.
- **학술 가치:** 논문(의학교육), 국시원 관심(특히 ④채점·신뢰도), 에듀테크 엑스포.

## 1. 핵심 설계 결정 — 팀이 이미 합의함 (반드시 반영)

| # | 결정 | 근거 |
|---|---|---|
| D1 | **채점 = 예/아니오 이분법.** 4단계 평가 X (AI 불가). 공감표현 = 키워드 감지("힘드셨겠어요") | 회의 Q3, 리스크2 |
| D2 | **체크리스트 표현 최적화:** "거나"→"또는", 복합질문은 개별 항목 분리, "컴퓨터가 알아들을 언어"로 변환 | 리스크2 |
| D3 | **신체진찰 = 텍스트 발화 체크만.** 실제 수행 검증 X ("청진기 대겠습니다" 발화했나만). AR은 고도화 이관 | 회의 Q2 |
| D4 | **사례 생성 = 기존 사례 변형.** 연령·성별·증상만 바꿔 유사사례 생성. 한 번에 다 업로드 후 AI가 필요한 것만 참고(RAG) | Q5, 결정사항 |
| D5 | **분류 체계 = 국가고시 48개 주증상 + 진료수행지침 51개 항목** 교차 | 회의 4 |
| D6 | **지도학습 데이터 = 초안 + 피드백 + 최종본** (= `사례개발피드백_*.hwp`가 바로 이것) | 회의 4 |
| D7 | **일반어 ↔ 의학용어 매칭 필요** ("가슴이 답답해요"=흉부 답답함). 별도 매핑 데이터는 아직 없음 | 회의 4, 카톡 |
| D8 | **저작권:** 원본 교과서·중국데이터 = 로컬만(비공개), 우리 사례는 공개가능하나 학습데이터는 비공개 | 회의 260611-3 |
| D9 | **GraphRAG = 초기 제외, 고도화 단계에 추가.** 초기는 하이브리드 벡터RAG(임베딩+BM25) | 김용하 결정 2026-06-18 |

## 4. 김용하(④채점·백엔드) 기술 방향 — 2026 최신 (조사 완료)

- **LLM:** Claude/GPT 상위 모델(품질 우선, 결제). 어댑터로 교체 가능.
- **음성(③):** Gemini Live API 등 **오디오 네이티브**(STT→LLM→TTS 체인 X), 한국어·저지연.
- **RAG:** **하이브리드**(임베딩 + BM25) + 리랭커. 임베딩 후보: Gemini Embedding 001(MTEB#1)/Voyage-3/BGE-M3.
- **벡터검색:** Qdrant/pgvector 또는 OpenSearch(ES계열, 키워드+벡터 동시).
- **오케스트레이션:** LangGraph(2026 프로덕션 표준, 상태·2라운드 루프·감사).
- **추적:** LangSmith. **스키마 강제:** Pydantic 구조화 출력.
- **GraphRAG:** 고도화 단계(D9).

## 5. 열린 질문 · 보류 · 리스크

### ⚠️ 꼭 정리할 것
- **역할 범위 [결정됨 2026-06-18]:** 팀 분담(채원우=①생성, 김용하=④채점)은 **명목상/서류상**. 실제로는 **김용하가 전체 end-to-end 단독 구현**. → 4 Agent 전부 김용하가 설계·구현. "뭘 먼저 깊게"의 우선순위가 중요(④채점 강점·국시원 가치에서 출발 검토).
- **샘플 케이스 확보:** 부산대 사례 2~3건(완성본) → 내 CPX 이해 + 생성 few-shot. 단톡 요청 중.

### 보류(리마인더)
- 프로젝트용 `CLAUDE.md`·`AGENT.md`·`MEMORY.md` = **전체 맥락 완전 파악 후** 생성(아직 X).
- `사례개발피드백` 표 본문(피드백 내용) = hwp5html로 추출 필요(② few-shot 만들 때).
- `기본진료수행지침`(798p 스캔본) = OCR 필요(분류 근거로 쓸 때).

### 이미 식별된 HIGH 리스크 (팀)
- **사례 생성 의학적 정확성·맥락 일관성**(최고난도): 남성에 산부인과력 X 등 → 다단계 검증 + 질병별 환자 프로파일.
- **채점 기준 모호성**(=내 영역): 복합질문, 공감표현 정량화 → D1·D2로 대응.

## 6. 작업 원칙 (사용자 지시)
