## 0. 프로젝트 한 눈에

- **무엇:** AI로 CPX(임상수행평가) ① 사례 생성 → ② 전문가 심사·개선 → ③ 가상환자 대화 → ④ 자동 채점·피드백 (4-에이전트 파이프라인 + 검증 하네스).
- **🔑 2026-07-01 대전환:** **온톨로지 지식그래프로 "근거 기반 생성"** — LLM이 아무렇게나가 아니라 구조화된 지식(원인–증상–질환) 안에서 근거를 갖고 사례 생성. **박정빈 교수 비전**(정본 = `ontology-plan.md §0`).
- **왜:** 사례개발 1건 15시간(교수 10명 1박2일)·채점 24명×8시간·학생 연습파트너 부족 → AI로 해소. 단 목적은 시장이 아니라 **투명·검증·재현**(오픈소스·부산대 무료·논문).
- **팀(역할):**
  - 임선주 교수 = PI(총괄·의학검증·발표·교신저자)
  - 이혜윤 교수 = 공동PI(사례검증·체크리스트 타당성·임상자문)
  - **박정빈 교수 = 온톨로지 방향 결정권자(👑)** — 그 뜻을 정확히 해석해 충실히 실행(반박·대안 아님)
  - 채원우 = **사례 생성 AI**(①) · 데이터 전처리
  - 조호영 = STT/TTS·시스템통합·UI·학사연동
  - 민성 = 온톨로지 협업
  - **김용하(나) = 채점 로직(④)·백엔드·DB·API·보안 (명목)** → **실제로는 전체 end-to-end 단독 구현**(§5)
  - 최수연 조교 = 데이터(사례 원본 제공)·행정
- **본질:** **학술 연구 프로젝트**(영리 아님). 논문 = 교수·연구진 주도, 김용하 = 학부생 개발 담당(저자 아님). 상세·불변 규약 = `AGENTS.md`.
- **일정 신호:** H2 검증 교수 파일럿 응답 대기(다음 게이트) · **7/2(목) 12시 온톨로지 미팅**(의대 3층 시뮬레이션센터).
