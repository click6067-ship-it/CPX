## 3. Teardown 차원 (분석 항목)
- **A. 기능 gap** — 전체 기능 vs 우리. (캡처: 사례선택·음성세션·채점리포트·신체진찰)
- **B. 파이프라인 추론** — 사례 어떻게 생성? 채점? (행동으로 추론: 사례 반복되나·의학적으로 맞나·채점이 의역/오류 잡나·일관적인가)
- **C. 사례 품질 audit** — 그들 생성 사례 2~3건 원문 → 의학정확성·근거·다양성(65세男흡연자 편향?)·red flag 처리.
- **D. 채점 품질 audit** — 일부러 틀린 수행 입력 → 채점이 엄밀/일관한가. 우리 하네스 대비.
- **E. UI/UX** — 약점 기록(호버 안됨 등). *우선순위 낮음(프론트는 나중).*
- **F. 비즈/포지셔닝** — 가격·의대협 채널·홍보(유튜브)·속도.
- **G. 종합** — 우리가 이기는 곳(근거·정량) / 따라잡을 곳(음성·신체진찰·속도·완성도) / 차용할 곳(짧은 홍보영상·빠른 반복).

## 6. Codex 검수 반영 (v2 전략) — 2026-06-18

> SOLO = "한국 CPX 연습에서 **가장 믿을 수 있는 채점·디브리핑**" — 보이는 루브릭 근거 + 로컬 검증.
