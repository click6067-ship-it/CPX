# 검증 Registry (validation registry) — 템플릿

> 모든 (미래) 논문 주장 → **재현 좌표**(데이터·코드·프롬프트·모델·결과)로 추적. H4-real 시작 시 채운다. (Codex 학술검수 권고)

## 주장 ↔ 재현 좌표
| # | 주장(claim) | 데이터버전 | commit | 프롬프트버전 | 모델/버전 | eval 스크립트 | 결과 artifact | 95% CI |
|---|---|---|---|---|---|---|---|---|
| 1 | (예: ④ AI채점이 임상교원 합의와 weighted kappa ≥ 0.6, 도메인별) | | | | | | | |
| 2 | (예: ③ 가상환자 과공개율 < X%, N회 반복) | | | | | | | |

## claim 언어 규칙 (과대주장 금지)
- ❌ "AI가 인간 채점자와 동등" / "완벽한 점수 예측"
- ⭕ "전문가 검토 후 사용 가능 수준" / "도메인별 일치도 κ=…(CI …)"
- smoke 결과(손-라벨·소표본)는 **검증 아님** — 반드시 "smoke/단위테스트" 명시.
