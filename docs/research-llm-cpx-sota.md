# 의학교육 LLM SOTA 조사 (CPX 프로젝트용)

> deep-research 결과 (2026-06-18). 6개 각도 · 27개 소스 fetch · 134개 주장 추출 → 25개 적대검증(3표 중 2표 반박이면 폐기) → **22 confirmed / 3 killed**.
> ⚠️ killed 항목 = "그럴듯하지만 검증 탈락" = **논문/주장에 쓰면 안 됨**.

---

## 1. ① 사례 생성 (Generation)

- **멀티에이전트 프롬프팅 > 단일 RAG GPT-4o.** 구조화된 멀티에이전트 구성 → **사용가능 OSCE 88%(14/16)** vs 단일 RAG는 31%가 수정 필요. (Wang 2024, JMIR `10.2196/82116`)
- **핵심 기법: "checklist-as-script" — 체크리스트를 사례 생성의 뼈대(seed)로 사용.** → 우리 `[붙임2]`·`실기문항저자점검표`를 생성 입력 골격으로 쓰는 게 정답.
- 일본어 임상 비네트 18개를 ChatGPT-4로 생성·검증한 사례 등 다국어 적용 다수 confirmed.
- ❌ **killed:** "단일 seed에서 perturbation으로 실패 transcript를 만들어 학생 수준 다양화" (arXiv 2604.08126) → 0-3 반박. **이 perturbation 기법은 신뢰 말 것.**

## 2. ② 검증 (Validation)

- **자동채점 = 이분법(예/아니오)이 강력.** 오픈모델 qwen3-32b가 **90.50%로 GPT-4o와 동급** → 팀 결정 D1 검증됨 + **채점은 오픈모델로 비용 절감 가능.**
- 전문가 HITL 수용률 ~87%, 단 **세밀 품질 ICC는 낮을 수 있음(0.18)** → 정밀항목 일치도는 어렵다는 현실. kappa는 0.9→엄격기준 0.66.
- **지배적 검증법 = 다수 전문가 리뷰(내용타당도).** 권장 순서: *전문가 간 일치도(interrater) 먼저 → 그다음 AI-전문가 일치도.*
- **Silver-labeling**(GPT-4o가 항목별 점수를 매겨 약지도 라벨 생성) confirmed → **인간 라벨 전 부트스트랩 기법.**
- ❌ **killed (2개·중요):**
  - "ChatGPT-3.5 생성 MCQ 변별도 미달" (PLOS) → 0-3
  - "GPT-4o 생성 MCQ가 인간 문항과 **심리측정적으로 동등**" (npj) → 1-2 폐기. → **"AI 문항이 인간과 동등하다"고 주장하지 말 것** (논문에서 위험).

## 3. ③ 가상환자 (Virtual SP)

## 4. ④ 자동채점·피드백 (Auto-scoring)

- 이분법 채점 강력(위). **오픈모델 경쟁력 있음.** Silver-labeling으로 라벨 부트스트랩.
- 읽을거리: arXiv `2501.13957`, `2503.05793`.

## 5. 함정 · 안전성

- **🚨 인구통계 편향 — 실측 confirmed:** LLM 생성 사례에서 **남성 79~84% 과대, 소수집단 0~1.5% 과소.** (PMID 40665345 / PMC12265265). → "65세 남성 흡연자" 문제는 진짜. **생성 단계에 인구통계 다양성 강제 필요.**
- **사실성: 하이브리드 RAG + 구조화된 4단계 self-check → 오류 3.3%로 억제** (MedTutor, arXiv 2601.06979 / PMC). → 우리 하이브리드 RAG + 자가검증 단계 정당화.

## 6. ⭐ 우리 파이프라인에 바로 적용할 실천 권고

1. **생성 = 멀티에이전트 + checklist-as-seed.** 단일 프롬프트 X. (생성→비평→수정 루프, 체크리스트가 뼈대) — LangGraph로 딱.
2. **생성에 인구통계 다양성 컨트롤 명시** (목표 분포에서 환자 성별·연령·배경 샘플링) → 80% 남성 편향 차단.
3. **근거 = 하이브리드 RAG + 4단계 self-check** → 사실 오류 ~3% 유지.
4. **채점 = 이분법**(팀 D1). 비용 위해 **오픈모델(qwen3-32b급) 검토** — 이분법 채점은 GPT-4o와 동급.
5. **검증 = 다수 전문가 일치도 먼저 → AI-전문가 일치도(ICC/kappa).** 정밀 ICC 낮을 수 있으니 **이분법 kappa** 위주. `사례개발피드백`을 silver-label로 부트스트랩.
6. **논문 주의: "AI 문항이 인간과 동등" 주장 금지**(검증 탈락). "전문가 검토 후 사용가능" 수준으로.
7. **가상환자(③)는 학술 공백 = 우리 기여 포인트** (일관성·정보 과공개 제어).
