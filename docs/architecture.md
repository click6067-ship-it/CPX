## 0.5 MVP 범위 + **2단계 게이트** ⭐ (Codex 1,2,3 + R2 new-flaw)

**v1 (먼저 끝낼 thin-vertical):** CpxCase 스키마 + 수동검증 gold 사례 5~10건 + 텍스트 가상환자 + 대화로그 이분법 채점 + **최소 교수 리뷰 UI(v1 포함)**
**연기(v2+):** 음성(Live API), 풀 멀티에이전트 생성, LangGraph 통합, 오픈모델 호스팅, 고도화 대시보드, **무거운 GraphRAG 검색 파이프라인**. *(단 온톨로지 **지식그래프[구조]** 는 §4.5로 채택됨 — 작은 흉통 그래프를 생성 뼈대로. 코퍼스 거대화 전까지 유보되는 건 GraphRAG의 검색 인프라이지 그래프 구조가 아님. `ontology-plan §1`.)*

**게이트 2종 (R2: 엔지니어링 vs 타당성 분리 — 한쪽 게이트로 둘 다 막으면 "너무 엄격 or 너무 느슨"):**
- 🟢 **엔지니어링 게이트(코딩 시작용·가벼움):** 스키마 + 손으로 만든 사례 1~2건 + 대화 fixture 5건 → ④채점 코딩·단위테스트 시작 가능. *(데이터 안 와도 여기까진 진행)*
- 🔴 **타당성 게이트(생성·하네스 "타당" 주장 전·엄격):**
  1. 부산대 gold 사례 **10건+**(교수 확정) + **gold 대화 transcript** + **항목별 인간 라벨**
  2. 도메인 분포 + 양/음성 prevalence + **최소 불일치(disagreement) 사례**
  3. `사례개발피드백` 초안→피드백 쌍 추출 · 진료수행지침 OCR 부분집합
  4. **한국어 lexicon = 첫 5~10사례에 한정**, 체크리스트 연결 동의어로 버전관리 *(별도 프로젝트화 금지)*

## 2. 데이터 계약 — `CpxCase` (스키마는 구조, 검증은 의미)

```python
class ChecklistItem(BaseModel):
    id: str
    domain: Literal["병력청취","신체진찰","환자교육","환자의사관계","사이시험"]
    question_open: str | None; question_closed: str | None
    patient_answer: str
    scoring_rule: str          # 관찰가능·행동위주·"또는"(D2)
    is_binary: bool = True     # 병력/진찰=이분법(D1). PPI는 별도(아래 §3-④)
    points: float

class CpxCase(BaseModel):
    title; chief_complaint; diagnosis
    function_weights: dict     # 병력40·진찰30·교육5·PPI20·사이5
    demographics: dict         # ★진단조건부 prior로 생성 (15)
    situation_instruction; vitals; examinee_task
    patient: Patient; present_illness: list[HistoryItem]
    family_hx; social_hx; obgyn_hx; past_hx
    checklist: list[ChecklistItem]
```
- **의미 검증기(필수, Pydantic 위에):** 타임라인 일관성 · red flag 누락 · 불가능 vitals · 남성↔산부인과 · 동의어 누락 · 과공개 → 적대 fixture로 테스트 (16)

## 3. 4 Agent 설계 (구현 순서 = ④→③→②→①)

### ④ 채점 (첫 구현 · 내 lane · 국시원 가치)
- 입력: 대화로그+체크리스트 → 출력: 항목별 도달/누락+영역점수+피드백
- **병력·진찰 = 이분법**(D1), 복합질문 "또는" 분리(D2), 누락 구체안내
- **PPI(20%) = 이분법 부적합** → 루브릭+인간 보조 또는 *정의된 미시행동* (10)
- 모델: **상위 먼저**, 로컬 kappa 통과 후 오픈 (9). 손으로 만든 사례+대화 fixture로 조기 테스트

### ③ 가상환자
- **항목별 유한상태 답변정책(FSM):** 허용사실 · 트리거구 · 거부/되묻기 규칙 · 반복질문 행동 · **과공개 점수** (7)
- 텍스트 우선, 음성=옵션(§1)
- 신체진찰=텍스트 발화 인식(D3)

### ② 심사
- 루브릭=`실기문항저자점검표` → Round1→수정안→Round2→**교수 HITL 확정**

### ① 생성 (마지막 구현 · 최난도)
- 멀티에이전트(생성→비평→수정), **checklist-as-seed** — 단 **"가설"**(부산대 10사례 로컬 수용률로 검증 전엔 법칙 아님) (4)
- **진단조건부 인구통계 prior** + 교수 예외규칙 (남성편향 차단하되 비현실 방지) (15)
- 하이브리드 RAG + **사례단위 실패테스트**(red flag·불가능역학·체크리스트불일치·한국어·PPI·교수거절사유) — "3%" 주장 폐기 (6)

## 4.5 온톨로지 지식그래프 레이어 ⭐ (v3 · 2026-07-01 박정빈 교수 pivot)

> **정본 = `ontology-plan.md`.** 여기는 아키텍처 상의 *위치*만 정한다.

**왜:** LLM 자유 생성은 필수요소 누락·날조 위험. **의학 지식을 온톨로지(원인–증상–질환)로 설계 → 지식그래프**로 만들어, 그래프가 사례 필수요소를 **강제** → "근거 갖고 생성"(박정빈 교수). *4-에이전트를 교체하는 게 아니라 **①생성의 입력 제약 + 사후 validator**로 얹는 레이어.*

**지식 3층 (경쟁 아닌 보완):**
- **지식그래프** = 원인·증상·질환·감별·red flag·체크리스트·과공개 규칙의 *관계·생성 뼈대*.
- **LLM wiki·롱컨텍스트** = 마크다운 정제지식(CPX 형식·루브릭) — 조각 검색이 아니라 통째로 읽힘(256k+, 로컬 35B Q4·TurboQuant).
- **RAG** = 큰 교과서(§4, 조연·축소 중) — 통째로 못 올릴 깊은 임상사실·그래프 빌드 재료.

**저장:** **YAML 정본**(`ontology/chest_pain.yaml`) + **Neo4j 거울**(한 방향 렌더, 시각화). YAML에서 편집·검증·git추적, Neo4j는 안 고침(동기화 리스크 회피). Neo4j 정본화 여부는 7/2 교수 확인.

**validator (결정론·LLM 0회):** 생성 사례를 온톨로지 카드와 대조 — **2-렌즈**("환자가 *가졌나* positive" vs "학생이 *선별했나* asked" 분리) · 6검사(required·red-flag·discriminator coverage·과공개·모순·checklist) · 한국어 부정문 처리. **표준 온톨로지(SNOMED 등)가 못 하는 CPX 고유 규칙.** `src/cpx/ontology_validator.py`, Codex 6R APPROVED·33테스트.

**생성 스캐폴드:** 질환 노드 선택 → 서브그래프 추출(required·discriminator·red_flag·checklist) → 제약 걸어 LLM 서사화(과공개 금지) → validator 검사 → 통과/실패 리포트. (baseline 자유생성 vs 온톨로지 제약 paired 비교; 데모에서 과공개 위반 2→0.)

**좁게 시작:** 흉통 1개 주증상(질환5=ACS+감별4, `review_status: draft`). Neo4j 전환·48주증상 확대·로컬 35B 주력화 **시점은 교수님 결정.**

**⚠️ 정직성:** 임상 내용 draft(교수 검증 전) · "근거 기반 생성" 효과는 *측정*(누락률·모순율·accept율) 후에만 주장.

## 5. 하네스 설계 (타당성 증명 = 엄밀하게)

| 하네스 | 질문 | 정답지 | 지표·임계(사전등록) |
|---|---|---|---|
| H1 사례품질 | 생성 사례 좋은가 | 점검표+교수 | 충족률, content validity |
| H2 평가신뢰도 | ②=전문가? | 사례개발피드백 | ICC/weighted kappa |
| H3 가상환자 | 일관·과공개? | N회 반복 | 일치율, **과공개율** |
| H4 채점타당도 | ④=인간? | 임상교원 2인 블라인드 | kappa(이분법) |

- **순서:** **전문가-전문가 일치도(floor) 먼저** → 합의점수 → AI-합의 일치도 (11)
- **사전등록(기본값, 모듈 빌드 시 확정):** 목표 **κ≥0.6(substantial)** · 전문가간 floor가 이보다 낮으면 *항목 재정의* · bootstrap 95% CI · 도메인별 최소 N(예: 항목당 ≥20 사례) · stop/kill 조건 (11, 가정3)
- **H4 2단계(R2 new-flaw):** 손 fixture = **smoke/단위테스트만**(깨끗해서 kappa 부풀려짐) → **kappa 타당성은 실제 VP·학생 transcript 후 재측정**
- **⚠️ silver-label = 스모크 only.** 타당성·교수보고·논문 금지 (순환검증) (12)
- **⚠️ 논문:** "AI=인간 동등" 주장 금지 (적대검증 탈락)

```

1 models.py (CpxCase + 의미검증기)         ← 계약
2 대화로그 fixture (손으로 5건)             ← ④ 단위테스트 재료

5 텍스트 ③ 가상환자(FSM 답변정책)
6 🔴타당성 게이트(§0.5) 충족 → H4-real: VP·파일럿 transcript로 kappa 재측정
```
