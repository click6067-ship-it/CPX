# 완성 체크 (Roadmap)

> 목표 = **검증된·투명한·오픈소스 CPX AI 시스템**(부산대 학생 무료) + 그 결과로 따라오는 **논문**(교수·연구진 주도; 김용하=학부생 개발 담당).

### H2 결과 후 단계
1. 결과로 **②B 보강** → 생성→검토→수정 **루프로 ① 생성 품질** 끌어올림(용하: "개선봇이 핵심").
2. **④채점 검증**(H4-real: 학생 transcript + 임상교원 라벨) — 논문 또 다른 핵심.
3. 지표/재현성/**오픈소스 패키징·투명성 산출물**(논문 재료).
4. **본검증**(사례·판정자 확대, 사전등록, 비열등성 δ) — 파일럿 신호 양호 시.

## ⬜ 빠진 것 (우선순위) — *"논문이 쓸 재료를 산출하는 빌드"* 기준
1. ★★ **실제 부산대 사례**(seed + gold) — **오늘 저녁 도착 예정** → 손-gold 교체, ①생성·실제검증 해금
2. ★ **H4-real 검증** — 실제 학생 transcript + 임상교원 라벨 → AI-인간 일치도(논문 핵심 결과)
3. ★ **① 생성 Agent** — 실제 사례 도착 후(타당성 게이트)
4. **지표 확장** — F1·precision 추가(현재 kappa·accuracy) ← 빠른 작업
5. **실제 사례 ingestion 변환기** — [붙임2]/원본 → CpxCase JSON (오늘 밤 사례용)
6. **오픈소스 패키징** — 라이선스(MIT?, MedQA처럼) · README · 재현성(프롬프트·데이터·시드 버전) · 공개 범위(비식별)
7. **투명성 산출물** — 검증 페이지(샘플 transcript · AI-인간 일치도 · 한계 명시)
8. **음성 완성도** — 교수 데모용(추후, 좋은 API)
9. **IRB 상태 확인** — 학생 데이터(제안서 명시) — 교수팀 영역
10. **④ 디브리핑 차원 확장** — 도달/누락 이분법 외에 **유도성 질문·전문용어 사용·개방형/폐쇄형·"다음에 고칠 문장"(리라이트)** (CPX 표준 채점기준; 메튜 리포트가 보여준 바, 교육효과·교수 설득력). 단 각 차원도 하네스로 검증.

## 🔑 정리 (놓친 부분, 정정판)
- 처음에 "논문 plan을 먼저 설계해야"라 했으나 — **정정: 너는 개발자, 논문은 교수팀 몫.** 빌드-주도가 맞다.
- 단 빌드가 **논문이 쓸 재료**(④번 지표·H4-real 검증데이터·재현성·투명성 산출물)를 *부산물로* 남기게 설계하면 됨. = 위 4·6·7번이 "논문 염두" 부분.
- 즉 시스템 완성도는 양호. **다음 = 실제 사례(저녁) 도착 → 투입.** 그 전까지 빠른 준비(F1/precision·ingestion 변환기).
