CPX 온톨로지 — 형식 비교: YAML vs Markdown

같은 흉통·ACS 지식을 두 형식으로. 왼쪽 = 구조(기계가 검증) · 오른쪽 = 산문(사람·LLM이 읽고 씀). 택일이 아니라 층마다 맞는 걸 쓴다.

⚠️ 임상내용 draft — 교수 검증 전 · 구조 시연용

🗂 YAML — 온톨로지(구조·규칙) validator가 기계로 검사 · git diff로 한 줄 추적 · Neo4j로 렌더

# 온톨로지 = 엔티티 + 관계 + 규칙 (코드가 읽음)
chief_complaint: chest_pain          # 주호소=흉통
disease:
  id: acute_coronary_syndrome        # ACS(엔티티)
  label_ko: 급성 관상동맥 증후군
  role: primary
  required_symptoms:                 # 반드시 포함 → 없으면 FAIL
    - id: chest_pressure
      keywords: [쥐어짜는, 누르는, 조이는, 답답]   # ← keyword-앵커
    - id: exertional_onset
      keywords: [계단, 오르막, 운동 중, 움직이면]
  discriminators:                    # 감별 단서
    - id: radiation_left_arm_jaw
      keywords: [왼팔, 어깨로 퍼지, 턱으로]
    - id: diaphoresis
      keywords: [식은땀, 땀이 났, 땀범벅]
  red_flags:                         # 응급 → 단순사례 금기
    - {id: hypotension, keywords: [혈압 낮, 어지러]}
    - {id: syncope,     keywords: [쓰러졌, 실신]}
  risk_factors: [흡연, 당뇨, 고혈압, 가족력]
  disclosure:                        # 과공개 규칙
    spontaneous: [chest_pressure]              # 먼저 말해도 OK
    disclose_if_asked:                         # 물어야만 답
      [radiation_left_arm_jaw, diaphoresis]
  source_id: TODO_교과서_또는_지침    # ⚠️ 근거 채우기 전 "근거기반" 주장 X
  review_status: draft

📄 Markdown — LLM wiki + 사례 사람·LLM이 읽고 씀 · "LLM wiki 처럼" · 교수님 "md로 사례변환"

## 급성 관상동맥 증후군(ACS) — 흉통  ⟨LLM wiki⟩

전형적으로 **쥐어짜는/누르는 흉통**이 **운동 중(계단·오르막)**
발생한다. **왼팔·턱 방사통**, **식은땀**이 동반되면 강하게 의심.
**저혈압·실신**은 응급 신호. 위험인자: 흡연·당뇨·고혈압·가족력.

> 학생이 **물어야만** 답할 것: 방사통·식은땀
> (환자가 먼저 흘리면 = 과공개 위반)

으로 내원. 20분 지속.

- (식은땀 물으면) "식은땀이 났습니다."
과거력: 흡연 20갑년, 당뇨.

🛡 예상 공격 — "왜 md 안 하고 yaml로 했냐?" · 방어

핵심 1줄: md를 안 쓴 게 아닙니다. wiki·사례는 md로 갑니다(교수님 지시대로). yaml은 온톨로지(기계 검증 대상)에만 썼습니다. 역할이 다른 두 층이라 둘 다 씁니다.

온톨로지를 yaml로 한 이유:

  1. 결정론 validator가 "필수요소 누락·과공개"를 LLM 0회로 기계 검사 → required: [A,B]는 코드가 읽지만, "필수 증상은 쥐어짜는 흉통과…"라는 산문은 못 뽑음.
  2. 재현성·감사(논문): 규칙이 구조화 → git diff 추적·검증 재현. (프로젝트 핵심가치)
  3. yaml → Neo4j 렌더 = 교수님이 원한 그래프 그대로.

"md로도 검증되잖아?" 반박: 산문 검증은 또 LLM을 불러야 함(비결정론·비용·순환검증 위험). yaml은 규칙이 명시적이라 확정·무료·감사가능.

정직한 양보(과대주장 방지): "온톨로지도 그냥 md로"를 교수님이 원하시면 가능 — 대신 기계검증을 포기(LLM검증으로 대체, 재현성↓). = 미팅 §7 확인 포인트.

같은 지식·두 형식. YAML=구조/규칙(검증·그래프), Markdown=지식/사례(읽기·서사). 임상내용 draft — 교수 검증 후 확정. 정본 설계 = docs/ontology-plan.md.
