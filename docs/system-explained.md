# CPX-AI 시스템, 완전 이해하기 (사람용 설명서)

> 처음 보는 사람도 이해하게 쉽게. 교수님 설명·복습용. (코드 정본은 src/, 설계는 architecture.md)

## 0. 한 문장
**"AI가 운영하는 CPX 연습 학원."** 시험 사례를 **쓰는 AI**, **검수하는 AI**, 환자를 **연기하는 AI**, **채점하는 AI** 4명 + 받치는 뒷단(도서관·통역사·품질검사실·관리자).

## 1. 큰 그림
```
              입력: "증상 + 진단"  (또는 기존 사례 변형)
                            │
  ┌───────────────── 만드는 쪽 (교수 대행) ──────────────────┐
  │  ① 작가(generator) ──초안──▶ ② 심사위원(reviewer)         │
  │       ▲                            │                      │
  │       └─ 고쳐써 ◀─ 미흡(Minor/Major)┘  ← 이 루프를 관리자(graph)가 자동
  │       (사서 rag가 교과서 근거를 대줌)      Accept ▼        │
  └──────────────────────────────────────────│──────────────┘
                                         확정 사례(CpxCase)
  ┌───────────────── 연습하는 쪽 (학생) ──────│──────────────┐
  │  ③ 환자배우(patient) ◀── 대화 ──▶ 학생                    │
  │       (묻는 것만 답·과공개 제어)      │ 대화록             │
  │  ④ 채점관(grader)+첨삭(debrief) ◀─────┘ →도달/누락·점수·빨간펜
  └──────────────────────────────────────────────────────────┘
                            │
  ──── 품질검사실(harness): "이 채점·심사 믿어도 되나?" ────
       정답지와 대조 → 정확도·F1·kappa / 적대테스트 / 환자 probe

  공통 뒷단: 계약서(models)·통역사(llm어댑터)·도서관(rag)·자재검수(quarantine·ingest)
```

## 2. 데이터가 흐르는 길
증상+진단 → **①작가**가 도서관(RAG) 교과서 근거를 보며 사례 작성 → **②심사위원** 검수(미흡하면 수정→재검수, 이 반복을 **관리자=LangGraph**가 자동) → **확정 사례** → **③환자배우**가 학생과 대화(묻는 것만) → **대화록** → **④채점관**이 체크리스트 대조해 도달/누락·점수, **첨삭** 빨간펜 → **품질검사실(하네스)**이 정답지와 대조해 숫자로 검증.

## 4. 왜 이렇게 지었나 (설계 사상)
1. 계약서 하나(models) — 제멋대로 X, 검증·교체 쉬움.
2. 통역사 어댑터(llm) — 싼 모델 검증→좋은 모델 1줄 교체.
3. 객관은 자로(이분법 ④)·주관은 사람(PPI 0/1/2+사람).
4. 베끼지 말고 근거로(RAG) · smoke(연습)≠validity(검증) 항상 구분.
5. 누수 방화벽 — 연습용/개발용/평가용 분리, 평가용 불혼합.
6. 전부 투명 — 프롬프트·데이터·지표 공개, worklog 기록(상용과 차별).

## 5. 머리에 담는 법
- 에이전트 4 + 받침 4 + 관리자 1.
- 동사로: **만든다(①②) · 연습한다(③④) · 믿는다(하네스).**
