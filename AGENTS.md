# AGENTS.md — CPX-AI (에이전트·Codex 공통 컨텍스트)

> 이 레포에서 일하는 모든 에이전트(Claude·Codex 등)가 **먼저 읽을** 맥락. 적대 검수 전 필독.

## ⚠️ 이 프로젝트의 본질 (오해 금지 — 가장 중요)
- **학술 연구 프로젝트.** 양산부산대학교병원 + 의대 교수(PI)와 공동. **영리·사업·스타트업 아님.**
- 목적 = **논문 + 검증(validation) + 투명성.** 돈·시장점유·유통·가격·홍보·first-mover = **무관.**
- 산출 = 부산대 내부 자료로 제작 → **부산대 학생에게 무료** · 작동방식 **전면 공개(오픈소스)** · 결과로 따라오는 **논문**(연구진·교수 주도 — **개발 담당 = 학부생 개발 담당, 논문 저자 아님**).
- **빌드-주도 + 논문 염두.** 개발자 관점: 작동·검증·투명 시스템을 빌드(교수 피드백→반복→학생 사용)하고 **논문은 결과로 따라온다.** 단 빌드가 *논문이 쓸 재료*(검증데이터·재현성·지표)를 산출하게 한다. "논문 먼저 설계" ❌ → **"논문 염두한 빌드"** ⭕. 순서를 엄격히 나누지 않음.
- 👉 **따라서 "B2C/구독/마케팅/시장/배포채널" 류 비즈니스 비평은 이 프로젝트엔 부적합.**
  적대 검수는 반드시 **학술 엄밀성 · 검증 타당성 · 재현성 · 의학적 안전성 · 과대주장 방지** 축으로만 할 것.

## 🔑 현재 최우선 설계 — 온톨로지 지식그래프 (지도교수, 2026-07-01)
> **모든 에이전트가 지금 정합을 맞춰야 할 최상위 설계.** 정본·상세 = `docs/ontology-plan.md §0`. 진입점 = `docs/blueprint.md`.

**설계 지시(정본):** 온톨로지를 적용 — **Neo4j 지식그래프로 구조화된 지식 내에서 LLM이 근거를 갖고 사례 생성**(아무렇게나 X). 숙제 7항목 = 엔티티 설계 · **keyword 기반 온톨로지** · **원인–증상–질환**(단순 graph 링크 아님) · **markdown CPX 변환** · **로컬 35B Q4·256k·TurboQuant** · **LLM wiki** · **Skill.MD** (상세 `ontology-plan §0`).

- **자세 = 교수 안을 정확히 해석해 충실히 실행(반박·대안 아님).** 최대 리스크 = 의도 오해. LLM wiki·지식그래프·RAG·로컬35B 전부 **교수의 단일 비전**(팀 발언=전달). 우리는 실행 + 결과 측정(측정 ≠ 저항).
- **온톨로지 = 4-에이전트의 추가 레이어**(①생성 제약 + 사후 결정론 validator), 교체 아님. 저장 = YAML 정본 + Neo4j 거울.
- **좁게 시작 = 흉통 1개.** Neo4j 전환·48주증상 확대·로컬 35B 주력화 **시점은 교수님 결정**(`ontology-plan §7`).
- ⚠️ 검수 시: 문서가 이 비전대로 갱신됐는지 + 과대주장(무측정 "근거 기반 생성") 없는지 함께 본다.

## 핵심 원칙
- **투명·검증·재현 가능** (모든 프롬프트·데이터·지표 공개). **과대주장 금지**("AI=인간 동등"·"완벽 채점" X) — 논문 정직성.
- 한국 CPX 자료가 **주 코퍼스**(영어교과서=의학사실 보조). 채점=이분법(PPI 분리). 하네스로 정량검증(**kappa·F1·precision**).
- 빌드 순서 = **④채점→③환자→②심사→①생성**(testable부터). 데이터 게이트 2단(엔지니어링/타당성).
- 실제 사례·전문가 라벨 전 = **smoke(손라벨)**, 타당성 주장 금지.
